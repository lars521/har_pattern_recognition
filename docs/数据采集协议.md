# 数据采集协议

## 1. 硬件配置

### 1.1 传感器与主控

| 器件 | 型号 | 用途 | I²C 地址 | 量程设置 |
| :-- | :-- | :-- | :-- | :-- |
| 加速度计/陀螺仪 | MPU6050 | 加速度、角速度 | 0x68 | 加速度 ±2g，陀螺仪 ±250°/s |
| 磁力计 | HMC5883L | 地磁场三分量 | 0x1E | 连续测量模式，增益 ±1.3Ga |
| 主控 | ESP32-DevKitC v4 | 数据采集与串口传输 | — | 时钟 240 MHz |

### 1.2 佩戴方式

- **佩戴位置**：腰部正前方（紧贴皮带或使用弹性绑带固定）
- **坐标轴定义**（以人体站立为参考）：
    - X 轴：水平向前（人体正方向）
    - Y 轴：水平向左（人体左侧）
    - Z 轴：垂直向上（重力反方向）
- **固定要求**：传感器壳体与身体无相对滑动，每次佩戴保持同一朝向


## 2. 采集参数

### 2.1 采样率

- 标称采样率：$f_s = 50$ Hz（通过 ESP32 定时器精确控制，允许 ±1 Hz 波动）
- 依据：满足采样定理（步频主频约 2–3 Hz，谐波至 10 Hz，50 Hz 足够）


### 2.2 数据格式

每行 CSV 记录包含以下字段（按顺序）：


| 字段 | 类型 | 说明 | 示例 |
| :-- | :-- | :-- | :-- |
| timestamp | int | ESP32 `millis()` 毫秒时间戳 | 12345678 |
| ax | int | 加速度 X 轴原始值（ADC） | -122 |
| ay | int | 加速度 Y 轴原始值 | 340 |
| az | int | 加速度 Z 轴原始值 | 16500 |
| gx | int | 陀螺仪 X 轴原始值 | 45 |
| gy | int | 陀螺仪 Y 轴原始值 | -12 |
| gz | int | 陀螺仪 Z 轴原始值 | 8 |
| mx | int | 磁力计 X 轴原始值 | -245 |
| my | int | 磁力计 Y 轴原始值 | 78 |
| mz | int | 磁力计 Z 轴原始值 | -312 |
| label | int | 活动标签（1–6） | 3 |
| subject_id | str | 被试代号 | "A" |

> 原始值暂不转换为物理单位，标定阶段再映射。

## 3. 被试方案

### 3.1 被试要求

- 人数：**至少 3 人**（记为 Sub_A, Sub_B, Sub_C）
- 年龄/性别/体型：鼓励多样化，但不做强制
- 健康状况：无严重运动障碍，能独立完成上下楼


### 3.2 采集次数

- 每人每类活动：**重复 3 次**
- 每次持续时长：**60 秒**（有效数据段，不含过渡）
- 总时长估算：3人 × 6类 × 3次 × 60s = 3240 秒 ≈ 54 分钟（实际加休息约 2–3 小时/人）


### 3.3 采集顺序

为减少疲劳，固定顺序如下（也可随机化）：

1. 静坐 → 2. 站立 → 3. 步行 → 4. 跑步 → 5. 上楼 → 6. 下楼

每类结束后休息至少 30 秒，跑步/上楼后休息 60 秒。

## 4. 标签与过渡处理

### 4.1 标签方法

- **方式**：串口输入数字 1–6 作为标签，采集脚本自动记录到 CSV 的 label 字段
- **时机**：开始每类活动前 2 秒输入标签，结束后输入 0 表示结束
- **同步**：标签输入时刻与数据流时间戳对齐


### 4.2 过渡段裁切

- 活动切换期间（如从静坐站起来的过程）数据不纯，必须剔除
- **裁切规则**：每类活动**前 2 秒**和**后 2 秒**的数据删除
- 实现方式：后处理脚本根据 label 变化点自动裁切


## 5. 数据划分策略（防泄漏）

### 5.1 基础划分

- **训练集**：Sub_A + Sub_B 的全部有效窗口
- **测试集**：Sub_C 的全部有效窗口


### 5.2 最终评估（LOSO）

将上述过程轮换三次，每次取不同被试做测试，最终报告平均性能：

- Fold 1: 训练 {A,B}，测试 {C}
- Fold 2: 训练 {A,C}，测试 {B}
- Fold 3: 训练 {B,C}，测试 {A}


### 5.3 严令禁止

- ❌ 将同一被试的数据同时放入训练集和测试集
- ❌ 在标准化、特征选择等步骤中，使用测试集的统计量


## 6. 采集流程（分步执行）

### 6.1 准备阶段

1. 传感器通过 USB 连接电脑，打开串口调试助手（或 Thonny）
2. 运行 ESP32 固件，确认 I²C 扫描到 `0x68` 和 `0x1E`
3. 运行 PC 端采集脚本 `collect.py`，设置好保存路径

### 6.2 正式采集

- 脚本等待输入标签 → 用户输入数字（1~6）
- 开始采集对应活动 60 秒，脚本自动记录 label 字段
- 60 秒结束后，脚本等待下一个标签

> 每类结束后脚本**自动**在 CSV 的 label 字段标记当前活动编号。

### 6.3 数据质量检查

- **每类结束后**，快速计算合加速度模长：

$$
|\mathbf{a}| = \sqrt{a_x^2 + a_y^2 + a_z^2}
$$
- **静止类**（静坐、站立）的合加速度模长应稳定在 **1g 附近**（约 9.8 m/s²，原始值需根据量程换算）。
- 若发现异常（数据全零、满量程、剧烈跳变），**立即重新采集该类**。


## 7. 文件命名与版本管理

### 7.1 原始数据文件命名

格式：`raw_SubjectX_YYYYMMDD_HHMMSS.csv`
示例：`raw_Sub_A_20250615_143000.csv`

### 7.2 目录结构

data/
├── raw/ \# 原始数据（标定前）
├── calibrated/ \# 标定后数据
└── windowed/ \# 窗口化后的特征矩阵

### 7.3 元数据记录

每个数据文件同目录下建议放置一个 `README.txt`，记录：

- 采集日期、被试、传感器固件版本
- 是否有异常情况
- 标定参数文件路径


## 8. 质量控制与避坑指南

| 问题 | 检测方法 | 纠正措施 |
| :-- | :-- | :-- |
| I²C 通信失败 | `i2c.scan()` 返回空 | 检查接线、上拉电阻、电源 |
| 静止时合加速度 ≠ 1g | 计算模长均值 | 重新做加速度计六位置标定 |
| 磁力计轨迹不是球 | 3D 散点图观察 | 重新做椭球拟合采集（多旋转） |
| 标签错位 | 检查 timestamp 与 label 对齐 | 采集时严格按协议输入 |
| 同一被试数据泄漏 | 检查 subject_id 分布 | 划分时按被试分组，不混合 |
